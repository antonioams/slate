    Introdução

    Bem vindo a documentação da API de integração da plataforma de ecommerce fornecida pela Vizir.

    Esta API segue o padrão REST/JSON e descreverá todos os recursos e endpoints necessários para a integração da plataforma de ecommerce com o ERP da Fazen.

    Este endpoint retorna a listagem de pedidos de acordo com os parâmetros enviados.

    HTTP Request

    GET http://api.fazen.com.br/v1/orders

    Query Parameters

    Parameter Default Description
    status processing Se enviado a API retornará apenas os pedidos que estão no status enviado.
    updated_at null Se enviado a API retornará apenas os pedidos que cujo valor do campo updated_at é maior ou igual a data e hora passada, caso contrário será retornado todos os pedidos independente da data e hora da ultima atualização.
    page_number 1 Se enviado iniciará a o retorno da listagem da API pa partir da página definida no valor do parâmetro page_number, caso contrário retornará a primeira página(valor padrão).
    page_size 50 Se enviado limitará a listagem de retorno da API para a qunatidade de registros definida no vbalor do parâmetro page_size, caso contrário retornará no máximo 50 registros (valor padrão).

    Requisita um pedido específico

    curl "http://api.fazen.com.br/v1/order/<id>"
  -H "Authorization: <authorization_key>"

    curl "http://api.fazen.com.br/v1/products"
  -H "Authorization: <authorization_key>"

    O comando acima retorna um JSON como o a seguir:

    {
  "count": 1,
  "products": [
    {
      "entity_id": "7292794",
      "sku": "133",
      "name": "Vacina x",
      "status": "0",
      "brand": "fabricante",
      "price": 122.00,
      "cost": 100.00,
      "stock": 0,
      "has_lead_time": "true",
      "lead_time_days": 5
    },
    {
      "entity_id": "7292795",
      "sku": "134",
      "name": "Vacina y",
      "status": "1",
      "brand": "fabricante",
      "price": 122.00,
      "cost": 100.00,
      "stock": 0,
      "has_lead_time": "true",
      "lead_time_days": 5
    }
  ]
}

    HTTP Request

    GET http://api.fazen.com.br/v1/products

    HTTP BODY

    Parameter Valor padrão Description
    status processing Se enviado a API retornará apenas os produtos que estão no status enviado.
    updated_at null Se enviado a API retornará apenas os produtos que cujo valor do campo updated_at é maior ou igual a data e hora passada, caso contrário será retornado todos os produtos independente da data e hora da ultima atualização.
    page_number 1 Se enviado iniciará a o retorno da listagem da API pa partir da página definida no valor do parâmetro page_number, caso contrário retornará a primeira página(valor padrão).
    page_size 50 Se enviado limitará a listagem de retorno da API para a qunatidade de registros definida no vbalor do parâmetro page_size, caso contrário retornará no máximo 50 registros (valor padrão).

    Criar produtos

    curl "http://api.fazen.com.br/v1/product"
  -H "Authorization: <authorization_key>"
  -X POST 
  -d "{json_obejct}"

    O comando acima deve enviar um objeto com o abaixo no body da requisição:

    {
  "sku": "144",
  "name": "vacina z",
  "brand": "Nome do Fabricante",
  "price": 122.00,
  "cost": 100.00,
  "stock": 0,
  "has_lead_time": "true",
  "lead_time_days": 5
}

    O comando acima retorna um JSON como o a seguir:

    {
  "entity_id": "7292799",
  "sku": "144",
  "name": "Vacina z",
  "status": "0",
  "brand": "Nome do Fabricante",
  "price": 122.00,
  "cost": 100.00,
  "stock": 0,
  "has_lead_time": "true",
  "lead_time_days": 5
}

    HTTP Request

    POST http://api.fazen.com.br/v1/product

    HTTP BODY

    Parameter Description
    sku SKU do produto a ser cadastrado, o SKU é um código único de identificação do produto.
    name Nome do produto.
    brand Marca do produto.
    price Preço do produto.
    cost Custo do produto.
    stock_qty Quantidade em estoque.
    has_lead_time Produto pode ser vendido em regime de lead_time.
    lead_time_days Quantidade de dias uteis para o lead_time do produto.

    Errors

    The API uses the following error codes:

    Error Code Meaning
    400 Bad Request – Your request sucks
    401 Unauthorized – Your API key is wrong
    403 Forbidden – The kitten requested is hidden for administrators only
    404 Not Found – The specified kitten could not be found
    405 Method Not Allowed – You tried to access has an invalid method
    406 Not Acceptable – You requested a format that isn’t json
    410 Gone – The kitten requested has been removed from our servers
    418 I’m a teapot
    429 Too Many Requests – You’re requesting too many requests!
    500 Internal Server Error – We had a problem with our server. Try again later.
    503 Service Unavailable – We’re temporarially offline for maintanance. Please try again later.
